# v2.8.3 (2026-04-26)

## 🔧 manifest: 加 `channelConfigs.wecom` + `channelEnvVars` 元数据

### 症状

openclaw 4.22 启动 wecom 插件时打 warn：

```
plugins.entries.wecom: plugin wecom: channel plugin manifest declares wecom
without channelConfigs metadata; add openclaw.plugin.json#channelConfigs so
config schema and setup surfaces work before runtime loads
```

### 根因

`openclaw/plugin-sdk/src/plugins/manifest.d.ts` 的 `PluginManifestChannelConfig` 要求声明了 channel 的插件在 manifest 里加 `channelConfigs.<channelId>` 描述 channel 级别的 config schema —— 这样 openclaw 在 runtime 加载之前就能用 schema 做 setup wizard / config validation / config UI 展示。

之前的 `openclaw.plugin.json` 只有：

```json
{
  "id": "wecom",
  "channels": ["wecom"],
  "configSchema": { "type": "object", "additionalProperties": false, "properties": {} }
}
```

**`configSchema` 是 `plugins.entries.wecom.config` 路径的 schema（用户实际是空的 `{enabled: true}`）**，跟 channel 真正的 config 路径 `channels.wecom.{accounts/bot/agent/kefu/...}` 是不同的两份 schema。openclaw 4.22 找不到第二份就 warn。

### 修法

**`openclaw.plugin.json`** 三处补齐：

1. **`channelEnvVars.wecom`**：声明 8 个常用环境变量，setup wizard 优先从 env 读
   ```json
   "channelEnvVars": {
     "wecom": [
       "WECOM_CORP_ID", "WECOM_AGENT_ID", "WECOM_AGENT_SECRET",
       "WECOM_CORP_SECRET", "WECOM_TOKEN", "WECOM_ENCODING_AES_KEY",
       "WECOM_BOT_ID", "WECOM_BOT_SECRET"
     ]
   }
   ```

2. **`channelConfigs.wecom.schema`**：mirror `src/config/schema.ts` 里 `WecomConfigInput` 的全部字段：
   - 顶层：`enabled` / `defaultAccount` / `mediaMaxMb` / `mediaDownloadTimeoutMs`
   - 兜底：`bot` / `agent` / `kefu`（顶层旧版兼容）
   - 多账户：`accounts: { [key]: { enabled, name, bot, agent, kefu } }`
   - 子系统：`media` / `network` / `routing` / `dynamicAgents`
   - bot 内部：`primaryTransport` (ws/webhook) / `dm` / `ws.{botId,secret}` / `webhook.{token,encodingAESKey,receiveId}`
   - agent 内部：`{corpId, token, encodingAESKey, agentSecret, corpSecret, agentId, dm}` (corpId/token/encodingAESKey 必填)
   - kefu 内部：`{corpId, corpSecret, openKfIds, webhook, dm}` (全部必填)
   - 用 `$ref` 让 accounts.<key>.bot 复用顶层 bot schema 避免重复

3. **`channelConfigs.wecom.label`** + **`description`**：给 setup UI 展示更友好

4. **`configSchema.additionalProperties: false → true`**：之前是 false 说明"plugins.entries.wecom.config 啥都不允许"，但插件并未读这层 config，改成 true 兼容未来扩展

### 不破坏

- 没改任何源代码；只动 `openclaw.plugin.json`
- 现有 `~/.openclaw/openclaw.json` 配置无需迁移（schema 是 `additionalProperties: true`）
- npm export / channel runtime 行为完全不变
- 只是让 openclaw 4.22 能在 runtime 之前读到 schema，启动 warn 消失

### 验证

```bash
# 升级到 v2.8.3 后看 ~/.openclaw/extensions/wecom/openclaw.plugin.json
grep -A 3 '"channelConfigs"' ~/.openclaw/extensions/wecom/openclaw.plugin.json
# 应看到 channelConfigs.wecom.schema 段

# 重启 openclaw 不再有 warn
```
