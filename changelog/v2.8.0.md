# v2.8.0 (2026-04-22)

## 🆕 微信客服（kefu）接入

第三条官方消息通道落地。Agent/Bot 之外，现在 OpenClaw 可以直接接管“企业微信客服”会话——外部客户在微信/视频号/企业对外小程序里发给客服账号的消息可以直接走到 OpenClaw 里回复。

### 架构一览

- **独立配置**：新增 `channels.wecom.accounts.<id>.kefu` 节点（`corpId` / `corpSecret` / `openKfIds[]` / `webhook.{token,encodingAESKey}`），`corpSecret` 可与 Agent 分开。
- **独立回调路径**：`/plugins/wecom/kefu`（推荐）与 `/plugins/wecom/kefu/<accountId>`，不与 Agent/Bot 冲突。企微回调只带 Token，插件内部调 `kf/sync_msg` 拉取，按 `open_kfid` 维度保留游标避免漏消息。
- **会话级去重**：LRU 记录 `msgid`，单账号 500 条、全局 2000 条；同一 `(accountId, openKfId)` 并发回调用 in-flight guard 合并。
- **源平面自动路由**：`WecomSourcePlane` 扩展出 `"kefu"`，`resolveWecomSourceSnapshot` 现在会返回 `kefuOpenKfId`。当用户先通过客服入向、Agent 回复时自动走客服出向（外部客户不会被同时收到 Agent 的企微内部私信）。
- **显式目标**：`wecom-kefu:<accountId>:<openKfId>:<externalUserId>` 和 `wecom-kefu:<openKfId>:<externalUserId>`（省略 accountId 时取默认账号）。

### 支持的消息类型

**入向（normalize → UnifiedInboundEvent）：**
text / image / voice / video / file / link / miniprogram / msgmenu / location / business_card / event（含 `enter_session` → `welcome` / 其它 → `kefu-session-event`）。

**出向（send_msg）：**
text（按 3500 字符切片，避开 4096 字节上限）、image/voice/video/file（由 `content-type` + 扩展名分类，走 `upload_media` 拿 media_id）、link（读 `payload.channelData.kefu.link`）。`toKefuText` 专用扁平化器把 markdown 降级成纯文本（因为 kefu `content` 字段不渲染 markdown）。

### 新增的文件

- `src/transport/kefu/{handler,normalize,cursor-store,outbound}.ts`
- `src/wecom_msg_adapter/kefu_text_adapter.ts`：`toKefuText(markdown, maxLength = 4096)`，把 OpenClaw 的富文本回复扁平成 kefu 渲染友好的纯文本（保留代码块内容、链接→`label (url)`、图片→`alt（图片：url）`、列表项→`•`、标题/引用/加粗/斜体/删除线/行内代码/HTML/分隔线全部剔除）。

### 配置示例

```yaml
channels:
  wecom:
    enabled: true
    accounts:
      default:
        kefu:
          corpId: "ww1234567890abcdef"
          corpSecret: "kefu-dedicated-secret"   # 可与 agent.corpSecret 独立
          openKfIds:
            - "wk_AAAbbbccc111"
          webhook:
            token: "xxx-token"
            encodingAESKey: "43位随机字符串"
```

### 联调要点

1. 企微管理后台配置客服回调 URL 到 `/plugins/wecom/kefu`（或 `/plugins/wecom/kefu/<accountId>` 显式指定账号）。
2. Token / EncodingAESKey 走 `channels.wecom.accounts.<id>.kefu.webhook.*`。
3. `corpSecret` 需要开启“读取消息”权限，最好独立建一个只给客服用的 Secret。
4. `openKfIds` 允许填多个（一个账号下多个客服号），每个 `open_kfid` 的 cursor 独立维护。

## ⚠️ 兼容性

- 纯 Bot/Agent 配置无需改动；`channels.wecom.accounts.*.kefu` 是可选项。
- `WecomSourcePlane` 新增 `"kefu"` —— 如果你在 `openclaw` 下游用到该类型的 narrowing，需要增加 case。
- `WecomSourceSnapshot.kefuOpenKfId?: string` 为可选字段，不会破坏既有消费者。
- `ReplyPayload.channelData.kefu.link` 为可选扩展字段，其他渠道/插件不读取即可忽略。
