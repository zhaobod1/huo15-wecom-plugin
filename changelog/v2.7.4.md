# v2.7.4 (2026-04-21)

## 🎨 格式升级

### 1. 自动抽取内联图片

正文里的 `![alt](url)` 不再依赖企微 markdown_v2 自行拉取（CDN 外链图片、跨域资源、hotlink 保护等场景经常渲染失败），插件在发送前主动下载上传得到 `media_id`，以独立 image 消息先于正文下发。

**执行流程：**
1. 发送前从文本中提取所有 `![alt](url)`
2. 逐张下载 → Agent API 调用 `upload media` → 以 image 消息类型下发
3. 剩余文本按原逻辑下发（markdown_v2 或 text）
4. 若某张图片上传失败，自动回退为内嵌 markdown 图片语法，不影响正文内容

**接入的路径：**
- ✅ Agent API 主动发送（`deliveryService.sendMedia`）
- ✅ Bot WS 主动发送（`handle.sendMedia` 带 `mediaUrl`）
- ⏸️ Bot Webhook `response_url`（单次消费限制，暂不接入多消息拆分）

### 2. `convertImages` 保留图片语法

`toWeComMarkdownV2` 不再把 `![alt](url)` 降级为 `[图片:alt](url)` 纯文本链接，改为规范化后保留图片语法，让企微 markdown_v2 有机会自行渲染（配合上面的抽取逻辑，常规场景以独立 image 消息优先）。

## 📦 安装优化

### 发布包裁剪至 ~900 kB（从 4.7MB）

新增 `.npmignore`，排除以下运行时无用内容：
- `assets/`：~3MB 的 README 说明图（GitHub 上仍能看到）
- `changelog/`：历史版本说明
- `SKILLS_CAL.md` / `SKILLS_DOC.md` / `UPSTREAM_*.md`：内部文档
- `*.test.ts` / `vitest.config.ts`：测试代码
- `.github/` / `scripts/`：CI 与开发脚本

**效果：**
- npm install 下载时间缩短 80%
- 降低网络抖动触发静默 `npm install failed` 的概率（2.7.3 部分用户反馈）

## 🧰 新增工具

### `src/wecom_msg_adapter/image_extractor.ts`

导出 `extractMarkdownImages(md)` 返回 `{ images, residualText }`，便于其它渠道/上游 Agent 定制扩展图片处理。

```typescript
import { extractMarkdownImages } from "@huo15/wecom/wecom_msg_adapter/image_extractor";

const { images, residualText } = extractMarkdownImages("Hello ![cat](https://x/c.png) world");
// images: [{ alt: "cat", url: "https://x/c.png" }]
// residualText: "Hello  world"
```

### `src/wecom_msg_adapter/image_fetcher.ts`

导出 `loadImageAsPayload(url)` 返回 `{ buffer, filename, contentType }`，统一处理 http(s) URL 与本地文件路径，自动推断 MIME。

## ⚠️ 兼容性

- 之前依赖 `[图片:alt](url)` 纯文本链接输出的自定义模板需要重新适配（大多数用户无感，adapter 默认行为改为企微原生渲染）
- `extractMarkdownImages` 只处理 `![alt](url)` 内联图片，不处理 `![alt][ref]` 引用式（原样保留）
