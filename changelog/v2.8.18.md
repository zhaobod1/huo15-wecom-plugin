# v2.8.18 (2026-05-02)

**chore-only release：注册 ClawHub plugin tag — 让不带版本号的 marketplace 安装能命中 plugin manifest**

## 背景

`openclaw plugins install @huo15/wecom`（**不带版本号**）走 ClawHub 协议解析（`clawhub:` schema）时，要求该 skill 在 ClawHub 上同时打了 `latest` 和 `plugin` 两个 tag —— `latest` 用来定位最新可用版本，`plugin` 用来声明"这是一个 OpenClaw 插件而不是普通 skill"，让 marketplace 能解析出 plugin manifest 走插件安装路径而不是文件投放路径。

之前 wecom 在 ClawHub 上**只有 `latest` tag，没有 `plugin` tag**——不带版本号的安装路径报"slug 找不到 plugin manifest"，必须用户手动写 `@huo15/wecom@2.8.17` 显式版本号才能装。

## 改动

### 1. `scripts/release.sh` — clawhub publish 加 `--tags latest,plugin`

[`scripts/release.sh:308`](scripts/release.sh:308)

- clawhub publish "$(pwd)" --version "$VERSION"
+ clawhub publish "$(pwd)" --version "$VERSION" --tags latest,plugin

零代码逻辑改动（`src/` 不动）。

## 风险评估（防"ghost manifest"事故）

之所以敢首次刷 plugin tag 还放心，是因为 wecom 历史所有版本（自 2.8.10 起）`compat.pluginApi` **全部是 ranged**（`>=2026.3.23`），不会重蹈 `@huo15/openclaw-enhance@5.7.9` "bare pluginApi 被冻结成 ghost manifest" 的覆辙：

> 当 ClawHub 把 plugin tag 指向某版本后，marketplace 会把那个版本的 manifest 缓存为"插件 entry"。如果该版本的 `compat.pluginApi` 是 bare（如 `"2026.3.23"` 精确匹配），任何更高版本 runtime 都装不上 → 用户看到 "no compatible plugin" → 实际是版本被钉死成 ghost。
>
> wecom 检查：所有 release 版本的 `pluginApi` 都是 `>=` range，runtime 升任意小版本都仍兼容。

详见 [CLAUDE.md §6.1](../../../CLAUDE.md#61-compatpluginapi-必须是-semver-range不能是裸版本)。
