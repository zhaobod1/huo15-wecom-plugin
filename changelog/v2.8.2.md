# v2.8.2 (2026-04-22)

## 🐛 修复图片+文字混合回复的 "COS 链接已过期"

### 症状

Agent 给 Bot 用户回复 "文字 + `![](COS/OSS 预签名 URL)`" 混合消息时，客户端显示 "COS 链接已过期 / 图片打不开"。纯图片回复不受影响。

### 根因

`sendTextViaBotWs` / Agent API 混合发送路径在**图片上传失败**时会执行降级：把原 `![alt](url)` 再写回 markdown 文本，让企微客户端"自己渲染"。

这个降级对普通 URL 还能用，但对**短 TTL 预签名 URL**（Tencent COS / 阿里 OSS / 图片生成工具的临时链接）是反效果——插件侧 fetch 已经失败的 URL，企微客户端拉同一 URL 一样失败，用户看到 "链接已过期"。

另外极端情况下 `extractMarkdownImages` 的严格正则可能漏匹配（URL 含 `)`、URL 中有空白、title 里有 `"`、跨行），漏掉的 `![](url)` 会随 markdown 文本发到客户端，同样触发 "过期"。

### 修法

1. **失败占位替换**：两条通道（Bot WS + Agent API）上传失败时不再把 URL 内联回 markdown，改成明文占位：
   ```
   ⚠️ 图片发送失败：<alt>（下载链接已过期，请重试）
   ```
   `buildImageFailurePlaceholder(alt, reason)` 对常见错误做归一化：
   - `expired / 403 / SignatureDoesNotMatch` → "下载链接已过期，请重试"
   - `timeout / ETIMEDOUT / AbortError` → "下载超时，请重试"
   - 超大小限制 → "超出大小限制"
   - 其它错误截单行 80 字符

2. **兜底 sanitize**：markdown 发出前最后一步 `sanitizeResidualImageMarkdown` 把任何残留 `![...](...)` 统一替换成占位——不管是什么原因漏网，都不让不可达 URL 送到客户端。

### 新增公共 API

- `buildImageFailurePlaceholder(alt: string, reason?: string): string`
- `sanitizeResidualImageMarkdown(text: string): string`

### 测试

`src/outbound.image-fallback.test.ts` 新增 11 个测试覆盖：
- placeholder 对空 alt / 过期 / 超时 / 未知错误的归一化
- sanitize 对单图、多图、URL 含不平衡括号、与普通 markdown 不误伤的行为

全量测试：160 passing（+11 新增），16 failing 与基线一致，无新增回归。

## ⚠️ 兼容性

- 纯向后兼容：只影响"上传失败时"的文本内容，成功路径行为不变。
- 如果你之前依赖 "上传失败时用户仍能看到原 URL 手动复制" 的降级行为，这个版本会显示 "图片发送失败" 占位。
